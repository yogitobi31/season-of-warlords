# 오디오 매니저 (Audio Manager)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-17
> **Implements Pillar**: Pillar 2 (눈에 보이는 성장) — 동료 합류 연출음이 성장의 청각적 확인

## Overview

오디오 매니저는 유랑단의 모든 BGM과 SFX 재생을 중앙에서 관리하는 Autoload Singleton이다. 두 가지 역할을 한다: (1) **BGM 관리** — 씬 전환 시스템의 요청에 따라 BGM을 페이드 아웃/인하고 트랙을 교체; (2) **SFX 재생** — 전투 타격음, 동료 합류 연출음 등 일회성 효과음을 요청 즉시 재생.

모든 오디오 요청은 `AudioManager` 단일 진입점을 통해 이루어진다. 씬 내 노드가 `AudioStreamPlayer`를 직접 생성하거나 재생하는 것은 금지 (단, UI 버튼 hover 등 극히 로컬한 효과음 제외).

**AudioBus 구조:**

| Bus | 용도 | 기본 볼륨 |
|-----|------|-----------|
| `Master` | 전체 볼륨 | 0 dB |
| `BGM` | 배경음악 | -6 dB |
| `SFX` | 효과음 | 0 dB |

> 버스 볼륨은 게임 설정(메인 메뉴/설정 UI)에서 플레이어가 조절 가능. 이 시스템은 버스 참조만 제공, 볼륨 저장은 세이브/로드 시스템 책임.

## Player Fantasy

해당 없음 — Foundation 인프라 레이어. 플레이어는 이 시스템의 존재를 인식하지 않는다. 플레이어가 느끼는 것은 이 시스템이 만드는 결과다: 새 지역에 들어설 때 음악이 자연스럽게 바뀌고, 동료가 합류하는 순간 청각적으로 특별한 무언가가 울린다.

## Detailed Rules

**R-1: 공개 인터페이스**
```gdscript
func play_sfx(sfx_id: String) -> void
func request_music_fade_out(duration: float = BGM_FADE_OUT_DURATION) -> void
func request_music_fade_in(scene_path: String, duration: float = BGM_FADE_IN_DURATION) -> void
func stop_music() -> void
```

**R-2: BGM 트랙 매핑**
씬 경로(`scene_path`)를 키로 BGM 트랙을 결정. 매핑 테이블은 외부 리소스(`audio_config.tres`)에서 로드.
```
"res://scenes/maps/forest.tscn" → "res://audio/bgm/forest_theme.ogg"
```
매핑에 없는 씬 경로 → BGM 없음 (조용함). 오류 아님.

**R-3: BGM 페이드 흐름**
씬 전환 시스템이 `request_music_fade_out()` 호출 → BGM 볼륨 트윈으로 0까지 감소 → `request_music_fade_in(scene_path)` 호출 → 새 트랙 로드 후 볼륨 0에서 정상까지 트윈 증가.

**R-4: SFX 재생**
`play_sfx(sfx_id)` 호출 시 풀에서 유휴 `AudioStreamPlayer`를 꺼내 재생. 재생 완료 시 풀 반환. `sfx_id`에 해당하는 리소스가 없으면 경고 로그 + 무시.

**R-5: BGM 중복 재생 방지**
현재 재생 중인 트랙과 동일한 트랙을 `request_music_fade_in()`으로 요청하면 트윈만 실행 (재시작 없음).

**R-6: SFX 풀 크기**
`AudioStreamPlayer` 풀 초기 크기 `SFX_POOL_SIZE`. 풀이 모두 사용 중이면 가장 오래된 SFX를 중단하고 재사용.

## Formulas

이 시스템에는 수학 공식이 없다. 볼륨 트윈은 Godot `Tween`이 처리하며, 이 시스템이 정의하는 것은 타이밍 상수와 풀 크기뿐이다.

**F-1: BGM 페이드 타이밍**
```
BGM_FADE_OUT_DURATION = 0.3초   ← 씬전환 시스템의 FADE_OUT_DURATION과 동기화
BGM_FADE_IN_DURATION  = 0.4초   ← 씬전환 시스템의 FADE_IN_DURATION과 동기화
```

| 상수 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `BGM_FADE_OUT_DURATION` | float | [0.1, 2.0]초 | BGM이 무음이 되는 시간 |
| `BGM_FADE_IN_DURATION` | float | [0.1, 2.0]초 | 새 BGM이 정상 볼륨에 도달하는 시간 |

**F-2: SFX 풀**
```
SFX_POOL_SIZE = 8
```

| 상수 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `SFX_POOL_SIZE` | int | [4, 32] | 동시 재생 가능한 최대 SFX 수. MVP 전투 규모(동료 3명 + 적 다수)에서 8이면 충분. |

## Edge Cases

**EC-1: `request_music_fade_in()` 중 또 다른 `request_music_fade_out()` 호출**
페이드 인 트윈 즉시 중단 → 현재 볼륨에서 페이드 아웃 시작. 씬 전환이 연속으로 빠르게 발생할 때(드물지만 가능).

**EC-2: 매핑에 없는 씬 경로**
R-2에 정의됨 — BGM 없음, 조용함. 오류 아님. 이전 BGM이 페이드 아웃된 뒤 침묵.

**EC-3: 오디오 파일 누락 (리소스 로드 실패)**
`play_sfx()` / `request_music_fade_in()` 시 파일이 없으면 경고 로그 + 무시. 게임 진행 차단 없음.

**EC-4: SFX 풀 고갈**
R-6에 정의됨 — 가장 오래된 SFX 중단 후 재사용. 전투 중 폭발적 SFX 요청(10명 동시 타격 등)에도 크래시 없음.

**EC-5: `stop_music()` 중 페이드 트윈 진행 중**
트윈 즉시 취소 → BGM 볼륨 0으로 강제 설정 → BGM 플레이어 정지.

**EC-6: 게임 시작 직후 BGM 없는 상태**
메인 메뉴 씬 경로가 `audio_config.tres`에 매핑되어 있으면 자동 재생. 매핑 없으면 침묵. 정상.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
없음 — Godot 4 빌트인 `AudioStreamPlayer`, `AudioServer`, `Tween`만 사용. 외부 데이터: `audio_config.tres` (씬→BGM 매핑 테이블).

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **씬 전환 시스템** | `request_music_fade_out()` / `request_music_fade_in(scene_path)` 호출. 씬전환 GDD에 이미 계약 명시됨. |
| **동료 합류 이벤트** | `play_sfx("companion_joined")` 호출로 합류 연출음 재생. |
| **체력/데미지 시스템** | `play_sfx("hit")`, `play_sfx("death")` 등 전투 SFX 호출. |
| **세이브/로드 시스템** | Master/BGM/SFX 버스 볼륨 값 저장·복원. 이 시스템은 현재 볼륨 값만 제공. |
| **메인 메뉴/설정 UI** | 볼륨 슬라이더가 `AudioServer.set_bus_volume_db()` 직접 호출. AudioManager 경유 불필요. |

## Tuning Knobs

| 변수 | 기본값 | 안전 범위 | 영향 |
|------|--------|-----------|------|
| `BGM_FADE_OUT_DURATION` | 0.3초 | 0.1–2.0초 | 씬전환 시스템 `FADE_OUT_DURATION`과 맞춰야 함. 어긋나면 BGM이 화면보다 먼저/늦게 사라짐 |
| `BGM_FADE_IN_DURATION` | 0.4초 | 0.1–2.0초 | 씬전환 시스템 `FADE_IN_DURATION`과 맞춰야 함 |
| `SFX_POOL_SIZE` | 8 | 4–32 | 작으면 동시 타격음 누락, 크면 메모리 낭비 |
| BGM Bus 기본 볼륨 | -6 dB | -20–0 dB | BGM이 SFX보다 너무 크거나 작지 않게 조정 |

> **중요**: `BGM_FADE_*_DURATION`은 씬 전환 시스템의 페이드 상수와 반드시 동기화. 두 값이 어긋나면 BGM과 화면 전환이 엇박자로 느껴짐.

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | `request_music_fade_out()` 호출 시 BGM 볼륨이 `BGM_FADE_OUT_DURATION` 내에 0으로 감소한다 | 수동 플레이테스트 |
| AC-2 | `request_music_fade_in(scene_path)` 호출 시 매핑된 트랙이 페이드 인되어 재생된다 | 수동 플레이테스트 |
| AC-3 | 매핑에 없는 `scene_path`로 `request_music_fade_in()` 호출 시 오류 없이 침묵 상태가 된다 | 단위 테스트 |
| AC-4 | `play_sfx()`로 동시에 8개 SFX 요청 시 모두 재생되고 크래시가 없다 | 단위 테스트 |
| AC-5 | 존재하지 않는 `sfx_id`로 `play_sfx()` 호출 시 경고 로그만 출력되고 게임이 멈추지 않는다 | 단위 테스트 |
| AC-6 | 씬 전환 중 BGM 페이드와 화면 페이드가 동기화되어 엇박자가 없다 | 수동 플레이테스트 |

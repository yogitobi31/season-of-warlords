# 수집/자원 시스템 (Collection & Resource System)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-17
> **Implements Pillar**: Pillar 1 (모인 사람들) — 퀘스트 아이템 수집이 동료 영입의 실질적 행동

## Overview

수집/자원 시스템은 맵에 배치된 수집 가능 아이템을 플레이어가 획득하는 메커니즘이다. 필드의 `PickupItem` 노드(`Area2D`)에 플레이어가 접근하면 자동으로 수집되어 인벤토리에 추가된다. 별도 입력 불필요 — 접촉 즉시 획득.

MVP에서 이 시스템의 역할은 단순하다: **퀘스트 조건 아이템을 필드에서 줍는다.** 희귀도, 드롭률, 장비 획득은 없다.

**PickupItem 노드 구조:**
```
PickupItem (Area2D)
├── CollisionShape2D    ← player_body 레이어 감지
├── Sprite2D            ← 아이템 아이콘
└── item_id: String     ← 획득할 아이템 ID (익스포트 변수)
    amount: int         ← 획득 수량 (기본 1)
```

## Player Fantasy

필드를 탐험하다 아이템을 발견하고 걸어가서 줍는 행동. 단순하지만 "퀘스트가 진행되고 있다"는 감각을 준다. 마지막 재료를 주울 때의 작은 만족감.

## Detailed Rules

**R-1: 자동 수집**
`PickupItem.area_entered(body)` — `player_body` 레이어 진입 시 `Inventory.add_item(item_id, amount)` 호출. 수집 후 `PickupItem` 노드 `queue_free()`.

**R-2: 적 사망 드롭**
적 `health_depleted()` 시 해당 적의 `drop_table`에 정의된 아이템을 적 위치에 `PickupItem`으로 인스턴스화. 드롭 테이블은 적 리소스 파일에서 정의.

**R-3: 에디터 배치 아이템**
맵 씬에 직접 배치된 `PickupItem` 노드. 씬 로드 시 존재, 수집 시 제거. 리스폰 없음 (MVP).

**R-4: 수집 피드백**
수집 시 `AudioManager.play_sfx("item_pickup")` 호출. 플로팅 텍스트(아이템명 + 수량) 표시는 Vertical Slice에서 추가.

## Formulas

수학 공식 없음. 드롭 수량은 `drop_table`에서 정수로 정의.

## Edge Cases

**EC-1: 동료가 PickupItem에 접촉**
`CollisionShape2D` 마스크를 `player_body`만으로 설정 — 동료/적 body는 감지 안 함. 플레이어만 수집.

**EC-2: 같은 프레임에 여러 PickupItem 접촉**
각각 독립적으로 `area_entered` 발생. 모두 수집됨. 정상 동작.

**EC-3: 인벤토리 용량 초과**
인벤토리 시스템 R-5에 정의됨 — MVP에서 용량 제한 없음. 항상 수집 성공.

**EC-4: 존재하지 않는 `item_id`**
인벤토리 `add_item()`이 경고 로그 + 무시. `PickupItem`은 사라짐 (아이템은 사라지지만 인벤토리엔 추가 안 됨). 설계 오류로 간주.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
- **인벤토리 시스템** — `Inventory.add_item()` 호출
- **아이템 데이터베이스** — `item_id` 유효성 (간접)
- **오디오 매니저** — `play_sfx("item_pickup")` 호출

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **퀘스트 상태 머신** | 인벤토리 `item_added` 신호를 통해 간접적으로 퀘스트 조건 갱신 |

## Tuning Knobs

| 변수 | 기본값 | 영향 |
|------|--------|------|
| `PickupItem` 감지 반경 | 20 px | 너무 작으면 수집 어려움, 너무 크면 의도치 않은 수집 |
| 드롭 수량 | 적 리소스 파일에서 설정 | 퀘스트 밸런스 직결 |

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | 플레이어가 `PickupItem`에 접촉 시 해당 아이템이 인벤토리에 추가된다 | 단위 테스트 |
| AC-2 | 수집 후 `PickupItem` 노드가 씬에서 제거된다 | 단위 테스트 |
| AC-3 | 동료가 `PickupItem`에 접촉해도 수집되지 않는다 | 단위 테스트 |
| AC-4 | 적 사망 시 `drop_table`에 정의된 아이템이 적 위치에 스폰된다 | 수동 플레이테스트 |
| AC-5 | 수집 시 SFX가 재생된다 | 수동 플레이테스트 |

# 동료 영입 퀘스트 (Companion Recruitment Quest)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-26
> **Implements Pillar**: Pillar 1 (모인 사람들) + Pillar 4 (작지만 진짜인 이야기) — 영입의 전체 경험

## Overview

동료 영입 퀘스트 시스템은 "대화 → 퀘스트 → 완료 → 합류"의 전체 영입 루프를 조율하는 Feature 레이어 시스템이다. 개별 시스템들(대화, 퀘스트 상태 머신, NPC 상태 관리, 동료 합류 이벤트)을 연결해 하나의 완결된 경험으로 만든다.

이 시스템이 직접 구현하는 것은 없다 — 다른 시스템들의 신호를 수신하고 올바른 순서로 이벤트를 발행하는 **오케스트레이터**다.

**전체 영입 루프:**
```
① NPC 대화 (RelationshipState.MET) → on_complete: "quest_X_started"
② QuestManager.start_quest("quest_X") → NPC 상태 → QUEST_ACTIVE
③ 플레이어가 퀘스트 조건 충족
④ QuestManager → quest_completed("quest_X")
   → NPCRegistry.set_state(npc_id, RelationshipState.COMPANION)  ← 퀘스트-상태-머신 R-5 처리
   → Inventory.remove_item(quest_items)                          ← 퀘스트-상태-머신 R-5 처리
⑤ 플레이어가 NPC에게 다시 말걸기 → COMPANION 상태 완료 대화 재생
⑥ 완료 대화 on_complete: "companion_X_join"
   → EventBus.emit_signal("companion_join_requested", companion_id)
⑦ 동료-합류-이벤트 처리 → 동료 파티 합류
```

## Player Fantasy

퀘스트를 마치고 그 사람에게 돌아가는 순간. "다 해결했어요"라고 말하는 대신 행동으로 보여준 결과. 그리고 그 사람이 "같이 가도 될까요?"라고 묻는 순간. 이 루프가 유랑단의 핵심 감정이다.

## Detailed Rules

**R-1: 퀘스트 완료 → NPC 상태 갱신**
`QuestManager.quest_completed(quest_id)` 신호는 퀘스트-상태-머신 R-5에서 `NPCRegistry.set_state(npc_id, RelationshipState.COMPANION)`을 직접 처리한다. 이 시스템에서 별도 NPC 상태 갱신 불필요. 완료 대화는 `RelationshipState.COMPANION` 키의 대화 리소스로 자동 선택됨 (대화 시스템 R-3).

**R-2: 완료 대화 → 합류 트리거**
완료 대화의 `on_complete: "companion_X_join"` 수신 시 → `EventBus.emit_signal("companion_join_requested", companion_id)`. 동료-합류-이벤트 시스템이 이 신호를 수신해 합류 처리.

**R-3: MVP 영입 퀘스트 3종**

| 동료 | 퀘스트 ID | 조건 | 보상 |
|------|-----------|------|------|
| 대장장이 | `quest_blacksmith` | 철광석 3개 수집 | 대장장이 합류 |
| 사냥꾼 | `quest_hunter` | 늑대 3마리 처치 | 사냥꾼 합류 |
| 약초사 | `quest_herbalist` | 특정 지역 방문 + 약초 2개 수집 | 약초사 합류 |

**R-4: 영입 순서 자유**
세 퀘스트는 병렬 진행 가능. 플레이어가 원하는 순서로 영입. 단, 특정 맵은 특정 동료 합류 후 접근 가능 (Vertical Slice에서 구현).

## Formulas

수학 공식 없음. 퀘스트 조건 수치는 퀘스트 데이터 파일에서 관리.

## Edge Cases

**EC-1: 퀘스트 완료 전 NPC에게 재대화**
NPC 상태가 `RelationshipState.QUEST_ACTIVE` — 진행 독려 대화 재생. 완료 대화 아님.

**EC-2: 동일 동료 중복 합류 시도**
`CompanionJoinEvent`에서 `PartyManager.is_recruited(companion_id)` 확인. 이미 합류 시 무시.

**EC-3: 여러 퀘스트 동시 활성**
허용. `QuestManager`가 독립적으로 추적. 세 퀘스트 동시 `ACTIVE` 가능.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
- **퀘스트 상태 머신** — `quest_completed` 신호 수신
- **대화 시스템** — `on_complete` 이벤트로 퀘스트 시작 / 합류 트리거
- **NPC 상태 관리** — NPC 상태 갱신

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **동료 합류 이벤트** | `trigger(companion_id)` 호출 |

## Tuning Knobs

퀘스트 조건 수치 (`data/quests/` 파일에서 조정):
| 퀘스트 | 조정 가능 값 | 영향 |
|--------|------------|------|
| `quest_blacksmith` | 철광석 요구 수량 | 수집 시간 / 난이도 |
| `quest_hunter` | 처치 요구 수 | 전투 시간 |
| `quest_herbalist` | 약초 요구 수량 | 탐험 범위 |

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | NPC 첫 대화 완료 시 퀘스트가 `ACTIVE` 상태로 시작된다 | 단위 테스트 |
| AC-2 | 퀘스트 조건 충족 후 NPC에게 말걸면 완료 대화가 재생된다 | 수동 플레이테스트 |
| AC-3 | 완료 대화 종료 시 `CompanionJoinEvent`가 발동된다 | 단위 테스트 |
| AC-4 | 세 퀘스트를 어떤 순서로 진행해도 각각 정상 완료된다 | 수동 플레이테스트 |
| AC-5 | 퀘스트 미완료 상태에서 NPC 재대화 시 진행 독려 대화가 나온다 | 수동 플레이테스트 |

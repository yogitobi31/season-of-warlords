# 플레이어 캐릭터 컨트롤러 (Player Character Controller)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-26
> **Implements Pillar**: Pillar 2 (눈에 보이는 성장) — 플레이어가 직접 조종하는 캐릭터가 팀의 중심

## Overview

플레이어 캐릭터 컨트롤러는 플레이어가 직접 조종하는 캐릭터의 이동, 공격, 상태를 관리하는 Core 레이어 시스템이다. 입력 매핑 시스템에서 입력을 받아 `CharacterBody2D.move_and_slide()`로 이동을 처리하고, 공격 입력 시 Hitbox를 활성화한다. 능력치 시스템에서 `move_speed`, `attack_damage` 등 수치를 참조한다.

이 시스템은 동료 AI 시스템과 의도적으로 다른 코드 경로를 사용한다 — 플레이어는 입력으로 제어되고, 동료는 AI로 제어되지만, 두 시스템 모두 동일한 `CharacterBody2D` 구조와 Hitbox/Hurtbox 계약을 공유한다.

**컨트롤러 상태 머신:**

```
IDLE ──공격 입력──► ATTACKING ──애니메이션 완료──► IDLE
  │                                                   │
  └──이동 입력──► MOVING ──입력 없음──────────────────┘
```

> 이동과 공격은 동시에 가능 (이동 중 공격). ATTACKING 중에도 `velocity`는 계속 갱신.

## Player Fantasy

플레이어가 캐릭터를 움직일 때 즉각적인 반응을 느껴야 한다. 입력과 화면 움직임 사이에 지연이 없고, 공격 버튼을 누르는 순간 캐릭터가 휘두른다. 혼자 싸울 때와 동료가 합류한 후 싸울 때의 차이가 이 컨트롤러를 통해 간접적으로 느껴진다 — 내가 움직이는 방향으로 팀 전체가 따라온다.

## Detailed Rules

**R-1: 이동**
매 `_physics_process()` 프레임에 `InputMapManager.get_move_vector()`로 방향 벡터 수신. `velocity = direction * move_speed`로 설정 후 `move_and_slide()` 호출.

**R-2: 공격**
`InputMapManager.is_action_just_pressed("combat_attack")` 감지 시 상태 → `ATTACKING`. `AnimationPlayer`로 공격 애니메이션 재생. 애니메이션의 특정 프레임에서 `HitboxArea2D.monitoring = true`, 프레임 종료 시 `false`. 애니메이션 완료 시 상태 → `IDLE`.

**R-3: `set_spawn_position(pos: Vector2)`**
씬 전환 시스템과의 계약 메서드. 호출 시 `global_position = pos`. 씬 전환 완료 이전(`PROCESS_MODE_DISABLED` 중)에도 위치 설정은 동작해야 함.

**R-4: 방향 전환**
이동 방향에 따라 스프라이트 `flip_h` 갱신. 마지막 이동 방향을 `facing_direction`으로 저장 — IDLE 상태에서도 공격 Hitbox 위치 결정에 사용.

**R-5: 사망**
체력/데미지 시스템이 `health_depleted` 신호 발행 → 컨트롤러가 수신해 상태 → `DEAD`. `DEAD` 상태에서는 입력 무시, `HurtboxArea2D.monitoring = false`, 사망 애니메이션 재생. MVP에서 사망 후 처리(리스폰)는 체력/데미지 시스템 책임.

**R-6: 공격 쿨다운**
`ATTACK_COOLDOWN`초 내 재공격 입력 무시. 연타 방지.

## Formulas

**F-1: 이동 속도**
```
velocity = move_direction.normalized() * move_speed
```

| 변수 | 출처 | 타입 | 설명 |
|------|------|------|------|
| `move_direction` | `InputMapManager.get_move_vector()` | Vector2 | 입력 방향 (정규화 전) |
| `move_speed` | 능력치 시스템 `stats.move_speed` | float | 픽셀/초 |

예시: `move_speed = 120`, 오른쪽 이동 → `velocity = Vector2(120, 0)`

**F-2: 공격 쿨다운**
```
can_attack = (current_time - last_attack_time) >= ATTACK_COOLDOWN
```

| 변수 | 기본값 | 타입 | 설명 |
|------|--------|------|------|
| `ATTACK_COOLDOWN` | 0.5초 | float | 공격 간 최소 간격 |
| `last_attack_time` | 0.0 | float | 마지막 공격 시각 (`Time.get_ticks_msec() / 1000.0`) |

> `attack_damage`는 이 시스템이 계산하지 않음. `attack_data` 딕셔너리에 `stats.attack_damage`를 담아 히트박스 시스템으로 전달.

## Edge Cases

**EC-1: 대각선 이동 시 속도 초과**
`move_direction.normalized()`로 방지. 대각선(Vector2(1,1))을 정규화하면 크기가 1로 유지됨.

**EC-2: ATTACKING 중 이동 입력**
허용. R-1 이동과 R-2 공격은 독립적으로 처리. 이동 중 공격 가능, 공격 중 이동 가능.

**EC-3: ATTACKING 중 재공격 입력**
R-6 쿨다운으로 방지. `ATTACK_COOLDOWN` 미경과 시 입력 무시.

**EC-4: DEAD 상태에서 입력**
R-5에 정의됨 — 모든 입력 무시. `_physics_process()`에서 상태 확인 후 조기 반환.

**EC-5: 벽에 밀착된 상태에서 이동 입력**
`move_and_slide()`가 자동 처리. 벽 방향 velocity 성분이 0이 되고 나머지 방향으로 슬라이드.

**EC-6: `set_spawn_position()` 호출 시 `PROCESS_MODE_DISABLED`**
R-3에 정의됨 — `global_position` 직접 설정은 process mode와 무관하게 동작. Godot 4 보장.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **입력 매핑 시스템** | `InputMapManager.get_move_vector()`, `is_action_just_pressed("attack")` |
| **능력치 시스템** | `stats.move_speed`, `stats.attack_damage` 참조 |
| **히트박스/충돌 감지** | `CharacterBody2D` 구조, `HitboxArea2D.monitoring` 제어 |

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **씬 전환 시스템** | `set_spawn_position(pos)` 계약 메서드 구현 (씬전환 GDD R-7) |
| **체력/데미지 시스템** | `health_depleted` 신호 수신해 `DEAD` 상태 진입. `hit_confirmed` 신호 수신해 피격 처리. |
| **동료 AI 시스템** | 플레이어 위치(`global_position`)를 추적 목표로 참조. |
| **게임 HUD** | 플레이어 상태(체력, 위치) 표시를 위해 참조. |

**양방향 계약 명시**
- 체력/데미지 시스템 GDD에 `health_depleted` 신호 명시 필요
- 씬 전환 GDD R-7에 `set_spawn_position` 계약 이미 명시됨 ✓

## Tuning Knobs

| 변수 | 기본값 | 안전 범위 | 영향 |
|------|--------|-----------|------|
| `move_speed` | 120 px/s | 80–200 px/s | 능력치 시스템에서 관리. 느리면 답답함, 빠르면 제어 어려움 |
| `ATTACK_COOLDOWN` | 0.5초 | 0.2–1.5초 | 짧을수록 연타 가능해 전투 빠름, 길수록 공격 무게감 있음 |
| Hitbox `CollisionShape2D` 크기 | 공격 애니메이션에 맞게 수동 설정 | — | 히트박스 GDD EC-6 참조. 시각적 모션과 1:1 검수 필수 |
| 공격 애니메이션 Hitbox 활성 프레임 | 수동 설정 | — | 너무 짧으면 판정 놓침, 너무 길면 불공정한 광범위 판정 |

> **조정 지침**: `move_speed`는 능력치 시스템에서 캐릭터마다 다를 수 있음. 이 컨트롤러는 값을 읽기만 하고 직접 정의하지 않음.

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | 이동 입력 시 캐릭터가 해당 방향으로 `move_speed` 속도로 이동한다 | 수동 플레이테스트 |
| AC-2 | 대각선 이동 시 속도가 수평/수직 이동과 동일하다 | 단위 테스트 |
| AC-3 | 공격 입력 시 Hitbox가 애니메이션 지정 프레임에서만 활성화되고 이후 비활성화된다 | 단위 테스트 |
| AC-4 | `ATTACK_COOLDOWN` 이내 재공격 입력은 무시된다 | 단위 테스트 |
| AC-5 | `set_spawn_position(pos)` 호출 시 캐릭터가 즉시 해당 위치로 이동한다 | 단위 테스트 |
| AC-6 | `DEAD` 상태에서 이동/공격 입력이 무시된다 | 단위 테스트 |
| AC-7 | 벽에 밀착된 상태에서 벽 방향 이동 입력 시 캐릭터가 슬라이드하며 막히지 않는다 | 수동 플레이테스트 |

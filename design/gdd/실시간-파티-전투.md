# 실시간 파티 전투 (Real-time Party Combat)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-17
> **Implements Pillar**: Pillar 2 (눈에 보이는 성장) — 이 시스템이 스노우볼 판타지의 실체
> **Risk Level**: HIGH — 모든 Core 시스템의 통합점, 설계 오류 시 전체 전투 느낌 붕괴

## Overview

실시간 파티 전투 시스템은 전투 조우(encounter)의 구조를 정의하고, 적 AI의 행동을 제어하며, 전투의 시작/종료 조건을 관리하는 Feature 레이어 시스템이다. 플레이어 컨트롤러, 동료 AI, 체력/데미지, 히트박스 시스템의 상호작용 위에서 동작하며, 이 모든 시스템이 하나의 전투로 느껴지도록 조율한다.

이 시스템이 직접 구현하는 것은 두 가지다: (1) **적 AI** — 플레이어를 향해 이동하고 공격하는 기본 행동; (2) **전투 조우 관리** — 맵의 적이 모두 처치되면 전투 종료 신호 발행.

**Pillar 2 실현의 핵심**: 동료 1명일 때와 3명일 때 화면이 눈에 띄게 달라야 한다. 적 수도 파티 규모에 비례해 증가.

**적 AI 상태 머신:**
```
IDLE ──플레이어 감지──► CHASING ──공격 범위 도달──► ATTACKING
  ▲                                                      │
  └──────────── 플레이어 사망 / 감지 범위 이탈 ◄──────────┘
```

## Player Fantasy

화면이 활기차야 한다. 동료가 없을 때는 내가 혼자 싸우지만, 3명이 함께하면 화면 곳곳에서 전투가 벌어진다. 적들이 나를 향해 달려오고, 동료들이 그 사이로 파고든다. 숫자가 아니라 눈으로 보이는 혼전. 그리고 마지막 적이 쓰러지는 순간의 조용한 해방감.

## Detailed Rules

**R-1: 적 AI — IDLE**
맵 배치 시 기본 상태. `DetectionArea2D`(반경 `ENEMY_DETECTION_RADIUS`) 내 `player_body` 또는 `companion_body` 진입 시 → `CHASING`. 타겟 우선순위: 플레이어 > 가장 가까운 동료.

**R-2: 적 AI — CHASING**
`NavigationAgent2D`로 타겟을 향해 이동. 공격 범위(`ENEMY_ATTACK_RANGE`) 도달 시 → `ATTACKING`. 타겟 사망 시 새 타겟 탐색. 감지 범위 내 타겟 없으면 → `IDLE`.

**R-3: 적 AI — ATTACKING**
이동 정지. `ENEMY_ATTACK_COOLDOWN`마다 공격 애니메이션 + Hitbox 활성화. 동료 AI와 동일한 Hitbox 패턴 사용 (히트박스 GDD R-2 계약).

**R-4: 전투 조우 (CombatEncounter)**
맵 씬의 `CombatEncounter` 노드가 해당 구역의 적 목록을 관리. 모든 적 `health_depleted()` 수신 시 → `combat_cleared(encounter_id)` 신호 발행.

**R-5: 파티 규모 연동 적 수**
`EnemySpawnZone`은 현재 파티 크기(`PartyManager.companion_count`)를 읽어 적 수 결정:
```
enemy_count = base_enemy_count + (companion_count * ENEMY_SCALE_PER_COMPANION)
```
동료가 늘수록 전장이 커진다.

**R-6: 플레이어 사망 시**
플레이어 `health_depleted()` → 전투 일시 중단 + 리스폰 처리. MVP에서 즉시 현재 맵 리스폰, 체력 전량 회복, 적 초기화.

## Formulas

**F-1: 적 수 스케일링**
```
enemy_count = base_enemy_count + (companion_count × ENEMY_SCALE_PER_COMPANION)
```

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `base_enemy_count` | 3 | 혼자일 때 기본 적 수 |
| `ENEMY_SCALE_PER_COMPANION` | 1 | 동료 1명당 추가 적 수 |

예시: 동료 3명 → `3 + (3×1) = 6`마리

## Edge Cases

**EC-1: 모든 동료 사망, 플레이어만 생존**
정상 전투 계속. 혼자 싸워야 하는 긴장 상황.

**EC-2: 플레이어 사망 후 리스폰 중 적 공격**
리스폰 처리 시작 시 플레이어 Hurtbox 비활성. 리스폰 완료 후 재활성.

**EC-3: 적이 맵 경계 밖으로 이동**
NavigationRegion2D 경계로 자동 제한. 경계 밖으로 나갈 수 없음.

**EC-4: 동료 합류 중 전투**
동료 합류는 전투 중에도 발생 가능. 합류 즉시 `FOLLOWING` 상태로 전투 참여.

**EC-5: `base_enemy_count = 0`인 조우**
`combat_cleared` 즉시 발행. 전투 없는 구역도 가능 (안전 구역).

## Dependencies

| 시스템 | 의존 내용 |
|--------|-----------|
| **플레이어 캐릭터 컨트롤러** | 플레이어 위치, 사망 신호 |
| **동료 AI 시스템** | 동료 전투 참여 |
| **체력/데미지 시스템** | `hit_confirmed`, `health_depleted` 신호 |
| **히트박스/충돌 감지** | 전투 판정 기반 |
| **오디오 매니저** | 전투 BGM, SFX |

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **동료 영입 퀘스트** | `combat_cleared` 후 퀘스트 조건 갱신 |
| **퀘스트 상태 머신** | `enemy_killed` 카운팅 |

## Tuning Knobs

| 변수 | 기본값 | 안전 범위 | 영향 |
|------|--------|-----------|------|
| `base_enemy_count` | 3 | 1–8 | 기본 전투 밀도 |
| `ENEMY_SCALE_PER_COMPANION` | 1 | 0–2 | 동료 증가에 따른 전장 확대 속도 |
| `ENEMY_DETECTION_RADIUS` | 180 px | 100–350 px | 적이 얼마나 멀리서 달려드는지 |
| `ENEMY_ATTACK_RANGE` | 45 px | 30–100 px | 적 근접 공격 범위 |
| `ENEMY_ATTACK_COOLDOWN` | 1.2초 | 0.5–3.0초 | 적이 플레이어보다 느릴수록 유리한 전투 |

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | 플레이어 접근 시 적이 감지하고 추격을 시작한다 | 수동 플레이테스트 |
| AC-2 | 동료 0명/1명/3명일 때 스폰 적 수가 각각 다르다 | 수동 플레이테스트 |
| AC-3 | 모든 적 처치 시 `combat_cleared` 신호가 발행된다 | 단위 테스트 |
| AC-4 | 플레이어 사망 시 현재 맵에서 체력 전량 회복 리스폰된다 | 수동 플레이테스트 |
| AC-5 | 동료 3명 + 적 6명 동시 전투에서 60fps를 유지한다 | 성능 테스트 |
| AC-6 | 혼자 싸울 때와 동료 3명일 때 화면 활기가 눈에 띄게 다르다 | 수동 플레이테스트 (주관 판정) |

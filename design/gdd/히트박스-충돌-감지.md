# 히트박스 / 충돌 감지 (Hitbox & Collision Detection)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-17
> **Implements Pillar**: Pillar 2 (눈에 보이는 성장) — 동료가 늘수록 전투 판정이 화면에서 실제로 작동해야 한다

## Overview

히트박스/충돌 감지 시스템은 유랑단의 모든 물리적 상호작용의 기반이다. 두 가지 역할을 한다: (1) **월드 충돌** — 캐릭터와 지형/장애물의 이동 충돌 처리 (`CharacterBody2D` + `move_and_slide()`); (2) **전투 판정** — 공격이 누구에게 맞았는지 결정하는 Hitbox/Hurtbox 구조. 이 두 역할은 서로 다른 Godot 레이어를 사용하며 명확히 분리된다.

모든 전투 판정은 `Area2D` 기반 Hitbox(공격 판정)와 Hurtbox(피격 판정)로 이루어진다. `PhysicsBody2D` 직접 충돌로 데미지를 계산하지 않는다. 이 구조 덕분에 이동 물리와 전투 판정이 독립적으로 튜닝 가능하다.

**Physics Layer 할당:**

| Layer | 이름 | 사용처 |
|-------|------|--------|
| 1 | `world` | 지형, 벽, 장애물 |
| 2 | `player_body` | 플레이어/동료 이동 충돌체 |
| 3 | `enemy_body` | 적 이동 충돌체 |
| 4 | `player_hitbox` | 플레이어/동료의 공격 판정 |
| 5 | `enemy_hitbox` | 적의 공격 판정 |
| 6 | `player_hurtbox` | 플레이어/동료의 피격 판정 |
| 7 | `enemy_hurtbox` | 적의 피격 판정 |

## Player Fantasy

해당 없음 — Foundation 인프라 레이어. 플레이어는 이 시스템을 직접 인식하지 않는다. 플레이어가 느끼는 것은 이 시스템이 보장하는 결과다: 공격이 시각적으로 닿는 순간 맞았다는 피드백이 오고, 벽에 막혀야 할 공격은 막히며, 내 팀원의 공격이 적에게만 닿는다.

## Detailed Rules

**R-1: 이동 충돌 (월드)**
모든 캐릭터(플레이어, 동료, 적)는 `CharacterBody2D`를 루트로 사용. 이동은 `move_and_slide()`로만 처리. Layer 1(`world`)과 각 진영의 body 레이어로 마스크 설정.

**R-2: Hitbox / Hurtbox 구조**
```
CharacterBody2D (캐릭터 루트)
├── CollisionShape2D          ← 이동 충돌체 (월드/body 레이어)
├── HurtboxArea2D             ← 피격 판정 Area2D (hurtbox 레이어)
│   └── CollisionShape2D
└── HitboxArea2D              ← 공격 판정 Area2D (hitbox 레이어, 평소 비활성)
    └── CollisionShape2D
```

**R-3: Hitbox 활성화 규칙**
`HitboxArea2D`는 공격 애니메이션의 특정 프레임에서만 `monitoring = true`. 공격이 끝나면 즉시 `monitoring = false`. 상시 활성 Hitbox 금지.

**R-4: 충돌 마스크 규칙**

| 노드 | Layer | Mask |
|------|-------|------|
| 플레이어/동료 CharacterBody2D | `player_body` | `world` |
| 적 CharacterBody2D | `enemy_body` | `world` |
| 플레이어/동료 HitboxArea2D | `player_hitbox` | `enemy_hurtbox` |
| 적 HitboxArea2D | `enemy_hitbox` | `player_hurtbox` |
| 플레이어/동료 HurtboxArea2D | `player_hurtbox` | `enemy_hitbox` |
| 적 HurtboxArea2D | `enemy_hurtbox` | `player_hitbox` |

**R-5: 데미지 신호 흐름**
`HitboxArea2D.area_entered(hurtbox)` 신호 발생 → Hitbox 소유자가 `hurtbox.owner`에게 `hit_confirmed(attack_data: Dictionary)` 신호 전달 → 체력/데미지 시스템이 수신해 처리. Hitbox는 데미지를 직접 계산하지 않는다.

**R-6: 다중 피격 방지 (Invincibility Frame)**
Hurtbox는 피격 후 `IFRAMES_DURATION`초 동안 `monitoring = false`. 같은 공격에 여러 번 맞는 것 방지.

## Formulas

이 시스템에는 수학 공식이 없다. 판정은 Godot Physics 엔진이 처리하며, 이 시스템이 정의하는 것은 타이밍 상수뿐이다.

**F-1: Invincibility Frame 지속 시간**
```
IFRAMES_DURATION = 0.5초
```

| 상수 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `IFRAMES_DURATION` | float | [0.1, 2.0]초 | 피격 후 무적 시간. 0이면 한 공격에 다단 피격 발생. |

**F-2: `attack_data` 딕셔너리 스펙**
```gdscript
attack_data = {
    "damage": int,          # 체력/데미지 시스템이 소비
    "knockback": Vector2,   # 넉백 방향+세기 (Vector2.ZERO = 넉백 없음)
    "attacker": Node,       # 공격자 참조
}
```

> 데미지 계산 공식은 체력/데미지 시스템(#9) GDD에서 정의. 이 시스템은 `attack_data`를 전달만 한다.

## Edge Cases

**EC-1: Hitbox가 여러 Hurtbox와 동시 접촉**
`area_entered`는 각 Hurtbox마다 개별 발생. 각각 독립적으로 `hit_confirmed` 신호 전달. 동일 프레임에 적 3명을 동시에 타격하는 것은 정상 동작.

**EC-2: 같은 공격이 같은 적에게 두 번 접촉**
R-6 Iframes로 방지. Hurtbox가 `monitoring = false` 상태면 `area_entered` 신호 자체가 발생하지 않음.

**EC-3: 플레이어/동료끼리 충돌**
이동 충돌체(body 레이어)는 같은 진영끼리 마스크하지 않음 → 서로 통과. 캐릭터가 겹쳐 보여도 이동에 방해되지 않음. MVP에서 동료 간 밀어내기 없음.

**EC-4: 적끼리 충돌**
동일. 적 body 레이어는 `world`만 마스크 → 적끼리 통과. AI가 밀집 이동해도 막히지 않음.

**EC-5: 캐릭터 사망 중 Hurtbox 활성**
체력/데미지 시스템이 사망 처리 시작 시 `HurtboxArea2D.monitoring = false`로 설정할 책임. 이 시스템은 사망 상태를 직접 알지 못한다.

**EC-6: CollisionShape2D 크기와 스프라이트 불일치**
의도적 허용. 픽셀 아트에서 시각적 크기보다 약간 작은 Hurtbox는 "관대한 판정"으로 플레이어에게 유리하게 느껴짐. 튜닝 문제이지 버그가 아님.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
없음 — Godot 4 빌트인 Physics 2D(`Area2D`, `CharacterBody2D`, `CollisionShape2D`)만 사용.

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **플레이어 캐릭터 컨트롤러** | `CharacterBody2D` + `move_and_slide()` 구조 사용. Hitbox 활성화 타이밍 관리 책임. |
| **동료 AI 시스템** | 동일 구조 사용. AI가 Hitbox 활성화 타이밍 관리. |
| **체력/데미지 시스템** | `hit_confirmed(attack_data)` 신호를 수신해 데미지 계산. Hurtbox의 Iframes 해제 타이밍 책임. |
| **실시간 파티 전투** | 이 시스템의 Layer/Mask 설정과 신호 구조 전제 위에 구축됨. |

**양방향 계약 명시**
- 체력/데미지 시스템 GDD에 `hit_confirmed(attack_data)` 수신 인터페이스 명시 필요
- 플레이어/동료 컨트롤러 GDD에 Hitbox 활성화 타이밍 관리 책임 명시 필요

## Tuning Knobs

| 변수 | 기본값 | 안전 범위 | 영향 |
|------|--------|-----------|------|
| `IFRAMES_DURATION` | 0.5초 | 0.1–2.0초 | 짧을수록 다단 피격 위험, 길수록 무적 구간이 길어 전투가 느슨해짐 |
| Hurtbox `CollisionShape2D` 크기 | 스프라이트의 ~80% | 60–100% | 작을수록 관대한 판정(플레이어에 유리), 클수록 엄격한 판정 |
| Hitbox `CollisionShape2D` 크기 | 공격 모션에 맞게 수동 설정 | — | 공격 범위 직접 결정. 애니메이션 프레임과 시각적으로 일치해야 함 |

> **조정 지침**: Hurtbox는 항상 스프라이트보다 약간 작게 유지 (플레이어 친화적 판정). Hitbox는 공격 애니메이션 키프레임과 1:1 검수 필수.

| AC-2 | 동일 공격에 같은 적이 `IFRAMES_DURATION` 내에 두 번 피격되지 않는다 | 단위 테스트 |

| AC-6 | Hitbox 비활성 상태(`monitoring = false`)에서는 겹침이 발생해도 신호가 없다 | 단위 테스트 |

# 대화 시스템 (Dialogue System)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-26
> **Implements Pillar**: Pillar 4 (작지만 진짜인 이야기) — 동료의 사연이 전달되는 유일한 통로

## Overview

대화 시스템은 플레이어와 NPC 간의 텍스트 대화를 표시하고 진행하는 Core 레이어 시스템이다. NPC 상태 관리에서 현재 NPC 상태를 읽어 적절한 대화 트리를 선택하고, 대화 완료 시 퀘스트 상태 머신에 이벤트를 전달한다.

MVP 범위를 명확히 제한한다: **보이스 오버, 복잡한 분기 대화, 대화 중 선택지 없음.** 대화는 순차적으로 진행되는 텍스트 박스다. 플레이어는 확인 버튼으로 다음 줄로 넘긴다.

**대화 데이터 구조:**
```gdscript
# 대화 리소스: 줄 배열
dialogue = [
    { "speaker": "마을 대장장이", "text": "이 칼... 녹이 슬어버렸네." },
    { "speaker": "마을 대장장이", "text": "광산에 철광석이 있다면 고칠 수 있을 텐데." },
]
```

## Player Fantasy

NPC와 대화할 때 "이 사람은 왜 여기 혼자 있는 걸까"가 자연스럽게 느껴진다. 텍스트 박스가 뜨는 순간 게임이 잠시 멈추고, 이 사람의 이야기에 집중하게 된다. 대화를 마쳤을 때 "이 사람을 도와야겠다"는 동기가 생겨난다.

## Detailed Rules

**R-1: 대화 시작**
플레이어가 NPC 근처에서 `interact` 액션 입력 → NPC가 현재 상태(`npc_state`)를 기준으로 대화 리소스 선택 → `DialogueManager.start_dialogue(dialogue_resource)` 호출. 대화 중 플레이어 이동/공격 입력 차단 (`InputMapManager` 통해).

**R-2: 대화 진행**
`confirm` 입력마다 다음 줄로 이동. 마지막 줄에서 `confirm` → 대화 종료. 텍스트는 타이핑 애니메이션(글자가 하나씩 나타남). `confirm` 입력 시 타이핑 중이면 전체 텍스트 즉시 표시 (스킵).

**R-3: 대화 리소스 선택 규칙**
NPC는 상태별 대화 리소스 딕셔너리를 보유:
```gdscript
dialogues = {
    RelationshipState.MET:          "res://data/dialogues/blacksmith_met.json",
    RelationshipState.QUEST_ACTIVE: "res://data/dialogues/blacksmith_quest_active.json",
    RelationshipState.COMPANION:    "res://data/dialogues/blacksmith_companion.json",
}
```
키는 `NPC-상태-관리.md`의 `RelationshipState` enum 값. 현재 `npc_state`에 해당하는 리소스 재생. 없으면 기본 대화("...") 출력.

**R-4: 대화 종료 이벤트**
대화 리소스에 선택적 `on_complete` 필드 — 대화 완료 시 퀘스트 상태 머신으로 전달할 이벤트 ID.
```gdscript
{ "on_complete": "quest_blacksmith_started" }   # 없으면 null
```

**R-5: DialogueManager Autoload**
`DialogueManager`로 등록. 대화 UI(CanvasLayer) 소유. 씬 전환 시에도 UI 레이어 유지.

## Formulas

수학 공식 없음. 타이핑 속도 상수만 정의.
```
TYPEWRITER_SPEED = 0.04초/글자   (초당 25글자)
```

| 상수 | 범위 | 영향 |
|------|------|------|
| `TYPEWRITER_SPEED` | 0.01–0.1초/글자 | 느릴수록 읽는 맛, 빠를수록 답답함 감소 |

## Edge Cases

**EC-1: 대화 중 NPC가 전투에 휘말림**
MVP에서 대화 가능한 NPC는 전투 구역 밖. 발생 안 함. 발생 시 대화 강제 종료 + `on_complete` 없이 닫힘.

**EC-2: 대화 중 씬 전환 요청**
`DialogueManager.is_active()` 체크. 대화 중 씬 전환 불가. 맵/지역 시스템이 이를 확인해야 함.

**EC-3: 존재하지 않는 `npc_state`에 해당하는 대화 리소스**
R-3에 정의됨 — 기본 대화("...") 출력. 오류 아님.

**EC-4: `on_complete` 이벤트 ID가 퀘스트 상태 머신에 없음**
경고 로그 + 무시. 대화는 정상 종료.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
- **NPC 상태 관리** — `npc_state` 읽기, 대화 리소스 선택 기준
- **입력 매핑 시스템** — `interact`, `confirm` 액션 수신

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **퀘스트 상태 머신** | `on_complete` 이벤트 전달로 퀘스트 시작/진행 트리거 |
| **동료 영입 퀘스트** | 영입 대화 트리거. 대화 완료 후 퀘스트 시작 이벤트 전달 |

## Tuning Knobs

| 변수 | 기본값 | 범위 | 영향 |
|------|--------|------|------|
| `TYPEWRITER_SPEED` | 0.04초/글자 | 0.01–0.1초 | 대화 읽기 속도감 |
| 대화 박스 위치 | 화면 하단 고정 | — | 전투 UI와 겹치지 않게 |

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | NPC 근처 `interact` 입력 시 해당 `npc_state`의 대화가 시작된다 | 수동 플레이테스트 |
| AC-2 | `confirm` 입력마다 다음 줄로 넘어가고 마지막 줄 후 대화가 닫힌다 | 수동 플레이테스트 |
| AC-3 | 타이핑 중 `confirm` 입력 시 전체 텍스트가 즉시 표시된다 | 수동 플레이테스트 |
| AC-4 | 대화 중 이동/공격 입력이 무시된다 | 수동 플레이테스트 |
| AC-5 | `on_complete` 있는 대화 종료 시 퀘스트 상태 머신에 이벤트가 전달된다 | 단위 테스트 |

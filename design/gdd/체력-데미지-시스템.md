# 체력/데미지 시스템 (Health & Damage System)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-26
> **Implements Pillar**: Pillar 2 (눈에 보이는 성장) — 전투의 긴장감과 위험이 실재해야 성장이 의미있다

## Overview

체력/데미지 시스템은 모든 캐릭터(플레이어, 동료, 적)의 체력 상태를 관리하고 피격 시 데미지를 계산하는 Core 레이어 컴포넌트다. 히트박스 시스템에서 `hit_confirmed(attack_data)` 신호를 수신해 데미지를 계산하고 `current_health`를 갱신한다. 체력이 0이 되면 `health_depleted` 신호를 발행하고 사망 처리를 위임한다.

이 시스템은 **컴포넌트**로 설계된다 — 플레이어, 동료, 적 모두 동일한 `HealthComponent` 노드를 자식으로 가진다. 캐릭터 종류마다 별도 구현이 없다.

**신호 흐름:**
```
히트박스 시스템
  hit_confirmed(attack_data)
        ↓
  HealthComponent
  데미지 계산 → current_health 갱신
        ↓
  health_changed(current, maximum)  →  전투 UI / HUD
  health_depleted()                 →  플레이어 컨트롤러 / 동료 AI / 적 AI
```

## Player Fantasy

플레이어가 적에게 맞았을 때 "아, 위험하다"는 긴장감이 있어야 한다. 동시에 회복과 전략으로 위기를 극복하는 만족감도 있어야 한다. 숫자가 아니라 화면의 체력바로 상태를 직관적으로 읽는다.

## Detailed Rules

**R-1: HealthComponent 구조**
모든 캐릭터의 자식 노드로 추가. 초기화 시 능력치 시스템에서 `stats.max_hp`를 읽어 `maximum_health` 설정. `current_health = maximum_health`로 시작.

**R-2: 피격 처리**
`hit_confirmed(attack_data)` 수신 시:
1. `final_damage` 계산 (F-1)
2. `current_health = max(0, current_health - final_damage)` 갱신
3. `HurtboxArea2D.monitoring = false` (Iframes 시작 — 히트박스 GDD R-6)
4. `health_changed(current_health, maximum_health)` 신호 발행
5. `current_health == 0`이면 `health_depleted()` 신호 발행

**R-3: Iframes 해제**
`IFRAMES_DURATION`(히트박스 GDD F-1: 0.5초) 후 `HurtboxArea2D.monitoring = true`. `HealthComponent`가 타이머로 관리.

**R-4: 사망 처리 위임**
`health_depleted()` 신호 발행 후 HealthComponent는 추가 작업 없음. 사망 애니메이션, 리스폰, 전투 종료 처리는 각 캐릭터 컨트롤러 또는 게임 매니저 책임.

**R-5: 회복**
`heal(amount: int)` 메서드. `current_health = min(maximum_health, current_health + amount)`. `health_changed` 신호 발행. MVP에서 회복 아이템/스킬이 없으면 미사용이지만 인터페이스는 정의.

## Formulas

**F-1: 최종 데미지 계산**
```
final_damage = max(1, attack_data["damage"] - target_defense)
```

| 변수 | 출처 | 타입 | 설명 |
|------|------|------|------|
| `attack_data["damage"]` | 공격자 능력치 시스템 `stats.atk` | int | 히트박스 시스템이 전달 |
| `target_defense` | 피격자 능력치 시스템 `stats.def` | int | 방어력. 0이면 데미지 그대로. |
| `final_damage` | — | int | 최솟값 1 보장 (방어력이 아무리 높아도 1 데미지) |

예시: `stats.atk = 15`, `stats.def = 5` → `final_damage = max(1, 10) = 10`

**F-2: 체력 갱신**
```
current_health = max(0, current_health - final_damage)
```
체력은 0 미만으로 내려가지 않는다.

**F-3: 회복**
```
current_health = min(maximum_health, current_health + heal_amount)
```
체력은 `maximum_health` 초과 불가.

## Edge Cases

**EC-1: `final_damage`가 `current_health`보다 큰 경우**
F-2로 처리됨 — `current_health = max(0, ...)`. 체력이 음수가 되지 않고 0으로 클램프. 즉시 `health_depleted()` 발행.

**EC-2: 이미 `DEAD` 상태에서 추가 피격**
`HealthComponent`는 `health_depleted()` 이후 `hit_confirmed` 수신을 무시. `is_dead` 플래그로 가드.

**EC-3: `maximum_health`가 0 이하로 초기화**
능력치 시스템 설계 오류. `HealthComponent._ready()`에서 `assert(maximum_health > 0)` — 개발 중 즉시 크래시로 발견.

**EC-4: `heal()` 호출 시 이미 최대 체력**
F-3으로 처리됨 — `current_health`가 변하지 않음. `health_changed` 신호는 발행하지 않음 (값 변화 없으므로).

**EC-5: Iframes 중 `heal()` 호출**
허용. 회복과 무적 시간은 독립적.

**EC-6: `target_defense >= attack_damage`**
F-1로 처리됨 — `final_damage = max(1, 음수 또는 0) = 1`. 방어력이 아무리 높아도 최소 1 데미지.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **능력치 시스템** | `stats.max_hp`, `stats.def` 참조 (능력치-시스템.md 권위 필드명) |
| **히트박스/충돌 감지** | `hit_confirmed(attack_data)` 신호 수신. `HurtboxArea2D.monitoring` 제어 |

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **플레이어 캐릭터 컨트롤러** | `health_depleted()` 수신 → `DEAD` 상태 진입 |
| **동료 AI 시스템** | `health_depleted()` 수신 → 동료 사망 처리 |
| **전투 UI / HUD** | `health_changed(current, maximum)` 수신 → 체력바 갱신 |
| **오디오 매니저** | 피격/사망 SFX 트리거 (`play_sfx("hit")`, `play_sfx("death")`) |

**공개 인터페이스 요약**
```gdscript
signal health_changed(current: int, maximum: int)
signal health_depleted()

func heal(amount: int) -> void
```

## Tuning Knobs

| 변수 | 기본값 | 안전 범위 | 영향 |
|------|--------|-----------|------|
| `stats.max_hp` (플레이어) | 80 | 50–300 | 능력치-시스템.md 권위값. 낮을수록 긴장감↑, 높을수록 여유로운 전투 |
| `stats.max_hp` (동료) | 능력치-시스템.md에서 정의 | 40–200 | 동료는 플레이어보다 약간 낮게 — 플레이어가 보호해야 하는 느낌 |
| `stats.max_hp` (적) | 30–60 | 10–200 | 적 유형마다 다름. 잡몹은 낮게, 엘리트는 높게 |
| `stats.def` (기본) | 0 | 0–20 | MVP에서 대부분 0. 방어력 시스템은 Vertical Slice에서 본격화 |
| `IFRAMES_DURATION` | 0.5초 | — | 히트박스 GDD에서 관리. 이 시스템은 참조만. |

> **조정 지침**: MVP 전투 밸런스의 핵심은 `max_health`와 `attack_damage` 비율. `max_health / attack_damage ≈ 5–8` 타격에 사망하는 게 적당한 긴장감 기준.

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | `hit_confirmed(attack_data)` 수신 시 `final_damage = max(1, damage - defense)`로 계산되고 `current_health`가 갱신된다 | 단위 테스트 |
| AC-2 | 피격 후 `health_changed(current, maximum)` 신호가 발행된다 | 단위 테스트 |
| AC-3 | `current_health`가 0이 되면 `health_depleted()` 신호가 발행된다 | 단위 테스트 |
| AC-4 | `current_health`는 0 미만으로 내려가지 않는다 | 단위 테스트 |
| AC-5 | `health_depleted()` 이후 추가 `hit_confirmed` 수신 시 무시된다 | 단위 테스트 |
| AC-6 | `heal(amount)` 호출 시 `current_health`가 `maximum_health`를 초과하지 않는다 | 단위 테스트 |
| AC-7 | `defense >= attack_damage`인 경우에도 최소 1 데미지가 적용된다 | 단위 테스트 |

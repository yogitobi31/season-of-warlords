# 씬 전환 시스템 (Scene Transition System)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-17
> **Implements Pillar**: Pillar 3 (팀이 곧 열쇠다) — 팀 구성에 따라 새로운 지역이 열리는 이동의 기반

## Overview

씬 전환 시스템은 유랑단의 모든 씬 로드/언로드를 중앙에서 관리하는 Foundation 인프라 레이어다. 두 가지 역할을 한다: (1) **씬 전환 실행** — 현재 게임 씬을 언로드하고 목표 씬을 비동기 로드하며, 전환 중에는 `PROCESS_MODE_DISABLED`로 다운스트림 시스템의 입력/업데이트를 차단한다; (2) **전환 피드백** — 페이드 아웃 → 로드 → 페이드 인 시퀀스를 통해 씬 전환이 갑작스럽게 느껴지지 않도록 한다. 모든 씬 전환 요청은 `SceneTransitionManager` (Autoload Singleton) 단일 진입점을 통해서만 이루어지며, 다른 시스템은 `SceneTree.change_scene_to_file()`을 직접 호출하지 않는다. 입력 매핑 시스템이 요구하는 process_mode 컨텍스트 분리의 실행 책임자다.

## Player Fantasy

해당 없음 — Foundation 인프라 레이어. 플레이어는 이 시스템의 존재를 인식하지 않는다. 플레이어가 느끼는 것은 이 시스템이 만드는 결과(씬 전환 시 화면 페이드, 새 지역의 로딩 없는 진입)이지, 시스템 자체가 아니다.

## Detailed Design

### Core Rules

**R-1: 단일 소유권**
`SceneTransitionManager` (Autoload Singleton)만 `get_tree().change_scene_to_packed()`을 호출할 수 있다. 다른 노드의 직접 호출은 Forbidden Pattern.

**R-2: 전환 요청 인터페이스**

```gdscript
func request_transition(
    scene_path: String,
    spawn_point_id: String = "",
    transition_type: TransitionType = TransitionType.DEFAULT,
    metadata: Dictionary = {}
) -> void
```

**R-3: TransitionType**
- `DEFAULT` — 페이드 아웃 → 비동기 로드 → 페이드 인. MVP 유일 사용 타입.
- `INSTANT` — 페이드 없음. `OS.is_debug_build()`일 때만 허용, 프로덕션 사용 금지.
- `CUT` — 검은 프레임 1프레임 삽입. 빠른 사망→재시작 등.

**R-4: metadata 계약**
새 씬 루트가 `_on_scene_ready(meta: Dictionary)` 메서드를 구현하면 자동 전달. 미구현 시 조용히 폐기.

**R-5: 전환 실행 흐름 (DEFAULT)**

```
request_transition()
│
├─ [Guard] state == IDLE? → 아니면 R-6 드롭
│
├─ [1단계 FADING_OUT]
│   현재 씬 루트 → PROCESS_MODE_DISABLED (입력/AI/물리 전부 차단)
│   CanvasLayer ColorRect: alpha 0→1 트윈 (FADE_OUT_DURATION초)
│   AudioManager.request_music_fade_out() 호출
│   트윈 완료 → [2단계]
│
├─ [2단계 LOADING]
│   ResourceLoader.load_threaded_request(scene_path)
│   _process()에서 load_threaded_get_status() 폴링 (메인 스레드)
│   SceneTransitionManager: PROCESS_MODE_ALWAYS 유지
│   로드 완료 → [3단계]
│
├─ [3단계 씬 교체]
│   load_threaded_get() → PackedScene
│   change_scene_to_packed(packed) — 이 함수만 이 위치에서 호출
│   await get_tree().process_frame → 새 씬 루트 참조 확보
│   스폰 위치 설정 (R-7)
│   metadata 전달 (R-4)
│   새 씬 루트 → PROCESS_MODE_DISABLED 유지 (페이드 완료 전까지)
│
└─ [4단계 FADING_IN]
    ColorRect: alpha 1→0 트윈 (FADE_IN_DURATION초)
    AudioManager.request_music_fade_in(new_scene_path) 호출
    트윈 완료:
      ColorRect.visible = false
      새 씬 루트 → PROCESS_MODE_INHERIT
      state → IDLE
      signal transition_completed(scene_path) 발행
```

**R-6: 중복 전환 요청 — 드롭**
전환 진행 중 새 `request_transition()` 호출은 무조건 무시 + 경고 로그. 취소-재시작 없음. 호출자는 `is_transitioning() -> bool`로 상태 확인.

**R-7: 스폰 위치 (SpawnPoint)**
새 씬의 플레이어 등장 위치 결정 우선순위:
1. `spawn_point_id` 이름과 일치하는 `Marker2D` 노드 (그룹: `"spawn_points"`, 이름: `SpawnPoint_{id}`)
2. 이름이 `SpawnPoint_DefaultSpawn`인 노드
3. 둘 다 없으면 `Vector2.ZERO` + 경고 로그

플레이어 캐릭터 컨트롤러는 `set_spawn_position(pos: Vector2)` 메서드 구현 필요 (플레이어 컨트롤러 GDD에 명시 예정).

**R-8: CanvasLayer 오버레이**
`SceneTransitionManager._ready()`에서 `CanvasLayer(layer=128)` + `ColorRect`를 Autoload 자신의 자식으로 추가. 씬 교체 시 Autoload는 유지되므로 오버레이가 씬 전환을 걸쳐 살아남음.

### States and Transitions

| 상태 | 설명 | 자신의 process_mode | 씬 루트 process_mode |
|------|------|-------------------|---------------------|
| `IDLE` | 요청 수락 가능 | ALWAYS | INHERIT |
| `FADING_OUT` | 페이드 아웃 중 | ALWAYS | DISABLED |
| `LOADING` | 비동기 로드 폴링 중 | ALWAYS | — (구 씬 언로드됨) |
| `FADING_IN` | 페이드 인 중 | ALWAYS | DISABLED → INHERIT (완료 시) |

```
IDLE ──request_transition()──► FADING_OUT ──완료──► LOADING ──완료──► FADING_IN ──완료──► IDLE
 ▲                                                                                           │
 └───────────────────────────────────── transition_completed 신호 발행 ────────────────────┘

※ FADING_OUT/LOADING/FADING_IN 중 request_transition() → 드롭 (R-6)
```

### Interactions with Other Systems

| 시스템 | 방향 | 인터페이스 |
|--------|------|-----------|
| **맵/지역 시스템** | 다운스트림 | `request_transition(scene_path, spawn_point_id)` 호출. `transition_completed(scene_path)` 신호 수신 후 BGM 전환/잠금 해제 체크. |
| **입력 매핑 시스템** | 계약 이행 | 직접 호출 없음. FADING_OUT 진입 시 씬 루트 `PROCESS_MODE_DISABLED`로 전환해 R-7 계약 이행. |
| **오디오 매니저** | 다운스트림 | `AudioManager.request_music_fade_out(duration)` / `AudioManager.request_music_fade_in(scene_path, duration)`. 오디오 매니저 미완성 시 null 가드로 방어적 처리. |
| **플레이어 캐릭터 컨트롤러** | 다운스트림 | `player.set_spawn_position(pos)` 호출. 플레이어 컨트롤러 GDD에 해당 메서드 계약 명시 필요. |

## Formulas

씬 전환 시스템에는 복잡한 수학적 공식이 없다. 타이밍 상수와 로드 진행도만 정의한다.

### F-1: 페이드 지속 시간

```
FADE_OUT_DURATION = 0.3초
FADE_IN_DURATION  = 0.4초
```

| 상수 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `FADE_OUT_DURATION` | float | [0.1, 1.0] 초 | 화면이 검어지는 데 걸리는 시간 |
| `FADE_IN_DURATION` | float | [0.1, 1.0] 초 | 새 씬이 밝아지는 데 걸리는 시간 |

페이드 아웃은 빠르게(0.3s), 페이드 인은 약간 느리게(0.4s) — 새 장소에 도착했다는 느낌을 준다.

### F-2: 로드 진행도

```
load_progress = progress_array[0]    ← ResourceLoader.load_threaded_get_status() 출력
```

| 변수 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `load_progress` | float | [0.0, 1.0] | 0.0 = 시작, 1.0 = 완료 |

`STATUS_LOADED` 반환 시 로드 완료. `STATUS_FAILED` 시 오류 처리 (Edge Cases 참조).

> **최소 표시 시간 없음**: MVP 씬이 작아 로딩이 즉시 완료될 수 있음. 페이드 아웃(0.3s)이 사실상 최소 전환 시간.

## Edge Cases

**EC-1: 로드 실패**
`STATUS_FAILED` 반환 시 — 페이드 인 없이 오버레이 유지, `push_error()` 출력, `state → IDLE` 복귀. 플레이어는 검은 화면을 보게 되지만 게임이 멈추진 않음. MVP에서 잘못된 `scene_path` 전달은 개발자 실수로 간주.

**EC-2: 전환 중 게임 종료 요청**
OS가 `notification(NOTIFICATION_WM_CLOSE_REQUEST)`을 보내면 전환 완료 대기 없이 즉시 `get_tree().quit()`. 데이터 손실은 세이브/로드 시스템 책임.

**EC-3: 동일 씬으로 전환**
`scene_path`가 현재 씬과 동일해도 전환 실행. 재시작/리스폰 용도로 유효.

**EC-4: spawn_point_id 미일치**
R-7에서 이미 처리됨 — `Vector2.ZERO` + 경고 로그.

**EC-5: 오디오 매니저 미초기화**
`AudioManager`가 null이면 null 가드로 무시, 전환은 계속 진행.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
없음 — Foundation 레이어의 최하단. Godot 4 빌트인 `ResourceLoader`, `SceneTree`, `Tween`만 사용.

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **맵/지역 시스템** | `request_transition()` 호출로 씬 이동 실행 |
| **플레이어 캐릭터 컨트롤러** | `set_spawn_position(pos)` 메서드 구현 필요 (R-7 계약) |
| **오디오 매니저** | `request_music_fade_out/in()` 메서드 구현 필요 |
| **입력 매핑 시스템** | `PROCESS_MODE_DISABLED` 계약 이행에 의존 (직접 호출 없음) |

**양방향 계약 명시**
- 플레이어 캐릭터 컨트롤러 GDD에 `set_spawn_position(pos: Vector2)` 반드시 명시
- 오디오 매니저 GDD에 `request_music_fade_out/in()` 인터페이스 반드시 명시

## Tuning Knobs

| 변수 | 기본값 | 안전 범위 | 영향 |
|------|--------|-----------|------|
| `FADE_OUT_DURATION` | 0.3s | 0.1–1.0s | 짧을수록 급격한 전환, 길수록 부드럽지만 답답함 |
| `FADE_IN_DURATION` | 0.4s | 0.1–1.0s | 짧을수록 새 씬이 툭 등장, 길수록 새 공간 도착감 강화 |
| `OVERLAY_COLOR` | `Color(0, 0, 0, 1)` | 검정 권장 | 흰색 플래시 등 연출 변형 가능, 남용 금지 |

> **조정 지침**: `FADE_OUT + FADE_IN` 합계가 1.0s 초과 시 플레이어가 전환을 지루하게 느낄 수 있음. MVP에서는 기본값 유지 권장.

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | `request_transition(path)`을 호출하면 페이드 아웃 → 씬 교체 → 페이드 인이 순서대로 실행된다 | 수동 플레이테스트 |
| AC-2 | 전환 중 `request_transition()` 재호출 시 두 번째 요청은 무시되고 경고 로그가 출력된다 | 단위 테스트 |
| AC-3 | `spawn_point_id`가 일치하는 `Marker2D`가 있을 때 플레이어가 해당 위치에 스폰된다 | 수동 플레이테스트 |
| AC-4 | 일치하는 `Marker2D`가 없을 때 `Vector2.ZERO`에 스폰되고 경고 로그가 출력된다 | 수동 플레이테스트 |
| AC-5 | 전환 중 씬 루트의 `process_mode`가 `DISABLED`이고, 전환 완료 후 `INHERIT`으로 복귀한다 | 단위 테스트 |
| AC-6 | 잘못된 `scene_path` 전달 시 게임이 크래시하지 않고 검은 화면 + 오류 로그 상태로 복귀한다 | 단위 테스트 |

# 맵/지역 시스템 (Map & Area System)

> **Status**: Done
> **Author**: Juwon + agents
> **Last Updated**: 2026-04-17
> **Implements Pillar**: Pillar 3 (팀이 곧 열쇠다) — 어디를 갈 수 있는지를 결정하는 세계 구조

## Overview

맵/지역 시스템은 게임 세계의 이동 구조를 정의하는 Core 레이어 시스템이다. 씬 전환 시스템 위에서 동작하며, 두 가지 역할을 한다: (1) **포털 트리거** — 플레이어가 이동 구역에 진입하면 씬 전환을 요청; (2) **지역 등록** — 어떤 씬이 어떤 지역인지, 어떤 포털이 어디로 연결되는지 정의.

MVP에서 맵은 3개. 각 맵은 독립적인 `.tscn` 파일이며 포털(`Area2D` 트리거)로 연결된다. 미니맵, 안개, 절차적 생성 없음.

**포털 구조:**
```
MapArea (씬 루트)
├── TileMap               ← 지형
├── SpawnPoint_Default    ← 기본 스폰 위치
├── SpawnPoint_{id}       ← 추가 스폰 포인트
└── Portal_{target}       ← Area2D 포털 트리거
    └── CollisionShape2D
```

## Player Fantasy

새 지역에 발을 들일 때마다 "여기는 어떤 곳일까"라는 기대감이 있어야 한다. 그리고 특정 동료가 있어야만 열리는 길 앞에서 "아, 저 사람을 먼저 찾아야겠구나"라는 직감이 생기면 이 시스템은 성공이다.

## Detailed Rules

**R-1: 포털 트리거**
`Portal_{target}` `Area2D`에 플레이어 body 진입 시 → `SceneTransitionManager.request_transition(target_scene, spawn_point_id)` 호출. `target_scene`과 `spawn_point_id`는 포털 노드의 익스포트 변수로 에디터에서 설정.

**R-2: MVP 맵 구성**

| 맵 ID | 씬 경로 | 설명 |
|-------|---------|------|
| `town` | `res://scenes/maps/town.tscn` | 시작 마을. 동료 후보 NPC 위치 |
| `forest` | `res://scenes/maps/forest.tscn` | 동료 1 퀘스트 지역 |
| `mine` | `res://scenes/maps/mine.tscn` | 동료 2 퀘스트 지역 |

**R-3: 스폰 포인트 명명 규칙**
씬 전환 GDD R-7 계약 이행:
- `SpawnPoint_Default` — 해당 맵의 기본 등장 위치
- `SpawnPoint_{portal_id}` — 특정 포털에서 진입 시 스폰 위치

예: `town.tscn`에서 forest 포털 진입 → `forest.tscn`의 `SpawnPoint_FromTown`에 스폰.

**R-4: 대화 중 포털 진입 차단**
포털 트리거 실행 전 `DialogueManager.is_active()` 확인. 대화 중이면 전환 요청 무시.

**R-5: 적 스폰 영역**
맵 씬 내 `EnemySpawnZone` 노드로 적 배치 정의. 씬 로드 시 자동 스폰. 동료 AI의 NavigationRegion2D도 맵 씬에 포함.

## Formulas

수학 공식 없음. 모든 맵 연결 정보는 에디터에서 포털 익스포트 변수로 설정.

## Edge Cases

**EC-1: 씬 전환 중 포털 재진입**
씬 전환 GDD R-6에서 처리됨. `is_transitioning()` 중 요청은 드롭.

**EC-2: 스폰 포인트 없음**
씬 전환 GDD R-7에서 처리됨 — `Vector2.ZERO` + 경고 로그.

**EC-3: 대화 중 포털 진입**
R-4에 정의됨 — 전환 요청 무시.

**EC-4: NavigationRegion2D 없는 맵에서 동료 AI**
동료가 이동 불가 → `FOLLOWING` 상태에서 제자리. 경고 로그. 모든 맵은 NavigationRegion2D 포함 필수.

## Dependencies

**이 시스템이 의존하는 것 (Upstream)**
- **씬 전환 시스템** — `request_transition()` 호출
- **대화 시스템** — `DialogueManager.is_active()` 포털 차단 확인

**이 시스템에 의존하는 것 (Downstream)**

| 시스템 | 의존 내용 |
|--------|-----------|
| **동료 AI 시스템** | NavigationRegion2D 제공 (맵 씬 내 포함) |
| **퀘스트 상태 머신** / **월드 조건 시스템** | `transition_completed` 신호 수신 후 지역 잠금 해제 체크 (Vertical Slice) |

## Tuning Knobs

| 변수 | 기본값 | 영향 |
|------|--------|------|
| 포털 `CollisionShape2D` 크기 | 맵 경계 너비 전체 | 너무 작으면 포털 진입 놓침 |
| 포털 위치 | 에디터에서 수동 배치 | 맵 가장자리에 자연스럽게 |

## Acceptance Criteria

| # | 조건 | 검증 방법 |
|---|------|-----------|
| AC-1 | 포털 진입 시 지정된 씬으로 전환되고 올바른 스폰 포인트에 등장한다 | 수동 플레이테스트 |
| AC-2 | 대화 중 포털 진입 시 씬 전환이 발생하지 않는다 | 수동 플레이테스트 |
| AC-3 | 3개 맵 간 양방향 이동이 모두 정상 동작한다 | 수동 플레이테스트 |
| AC-4 | 새 맵 로드 시 동료 AI의 NavigationAgent2D가 경로를 정상 계산한다 | 수동 플레이테스트 |
